# [Technical Guide] 파이썬 전문 퀴즈 시스템 (Console Quiz Game)

이 프로젝트는 단순히 기능만 작동하는 것을 넘어, **데이터의 안전**과 **코드의 구조적 견고함**을 최우선으로 설계된 전문가급 퀴즈 시스템임.

---

## Ⅰ. 핵심 기술 및 설계 원리 (쉬운 용어 설명)

### 1. 원자적 쓰기 (Atomic Write)
- **무엇인가?**: 파일을 저장할 때 "전부 저장되거나, 아예 안 되거나" 둘 중 하나만 선택하는 방식임.
- **왜 쓰는가?**: 저장 도중 컴퓨터가 꺼졌을 때, 파일이 반만 저장되어 내용이 깨지는(0바이트) 대참사를 막기 위함임.
- **어디에 있는가?**: `game.py`의 `save_state()` 메서드. `.tmp`라는 임시 파일에 먼저 내용을 다 쓰고, 완벽할 때만 `os.replace()`로 원본과 갈아치움.

### 2. 로그라이크형 무결성 설계 (Procedural Design)
- **무엇인가?**: 게임의 한 판(세션)이 끝나면 이전의 지저분한 상태를 남기지 않고 완벽하게 초기화하는 방식임.
- **왜 쓰는가?**: 이전 게임의 점수가 다음 게임에 영향을 주거나, 메모리가 꼬이는 현상을 막아 시스템을 항상 '새것'처럼 유지하기 위함임.
- **어디에 있는가?**: `game.py`의 `play_quiz()`와 `update_history()`. 각 플레이 결과를 독립적인 히스토리 객체로 관리함.

### 3. 전체 방어막 (Unified Exception Shield)
- **무엇인가?**: 프로그램 전체를 감싸는 거대한 '최후의 비상망'임.
- **왜 쓰는가?**: 사용자가 `Ctrl+C`를 눌러 강제료 종료하더라도, 그 찰나의 순간에 데이터를 저장소로 피신시키기 위함임.
- **어디에 있는가?**: `main.py`의 `main()` 함수 내부 `try-except` 구문.

## Ⅱ. 파일별 역할 및 상세 구현 위치

### 1. [데이터 설계도] quiz.py
- **역할**: 퀴즈 하나가 가져야 할 정보(질문, 보기, 정답)를 정의함.
- **구현 특징**: `from_dict()`를 통해 하드디스크의 데이터를 파이썬 객체로 조립함.

### 2. [입력 필터링] utils.py
- **역할**: 사용자가 숫자 대신 문자를 치거나, 엉뚱한 값을 넣는 것을 입구에서 컷트함.
- **구현 특징**: `get_valid_input()` 함수 하나로 시스템 내의 모든 숫자 입력을 정제함.

### 3. [시스템의 두뇌] game.py
- **역할**: 퀴즈 출제, 점수 계산, 파일 저장/불러오기를 총괄함.
- **독작 특징 (Self-Seeding)**: 파일이 없으면 **코드로 미리 준비된 나라별 수도 퀴즈**를 자동으로 불러옴. (22번 줄 근처 `load_state` 로직)

### 4. [입력 환경 개선] main.py
- **역할**: 메뉴를 출력하고 전체 게임을 구동함.
- **CJK(한글) 최적화**: `import readline`을 통해 한글 입력 시 잔상이 남거나 백스페이스가 씹히는 고질적인 터미널 문제를 해결함.

## Ⅲ. 실행 및 관리 가이드

### 1. 실행 방법
```bash
# 최상위 디렉토리에서 아래 명령어로 통합 실행
uv run quiz-game/main.py
```

### 2. 데이터 관리
- 모든 데이터는 `quiz-game/state.json`에 저장됨.
- 초기 로직에 의해 첫 실행 시 **대한민국 수도 퀴즈 실습 세트**가 자동으로 로드됨.
