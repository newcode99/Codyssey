# [Ultimate Project] 파이썬 전문 퀴즈 시스템 (Console Quiz Game)

본 문서는 파이썬 객체지향(OOP)의 근본 원리와 견고한 설계 철학을 집대성한 기술 명세이자 학습 일지임.

---

## Ⅰ. 객체지향(OOP) 가이드 및 학습 노트

### 1. Class: 설계도와 인스턴스
- **정의**: 데이터(멤버)와 기능(메서드)을 하나로 묶은 설계 도면. 클래스(Class)는 객체(Object)를 만들기 위한 **설계도나 틀**임.
- **클래스 vs 클로저**:
    - **클래스**: 설계도를 만들고 그 틀로 객체를 찍어내어 상태(self)를 영구적으로 기억함.
    - **클로저**: 함수 안에 변수를 숨겨두고, 그 변수를 사용하는 내부 함수만 꺼내어 쓰는 방식.
- **핵심 역할**:
    - 캡슐화 및 정보 은닉을 통해 외부의 부당한 조작으로부터 데이터를 보호함.
    - 동일 구조의 객체를 다수 생성하여 메모리를 효율적으로 관리함.

### 2. 클래스 메서드 (@classmethod) 동작 규칙
파이썬의 데코레이터 유무에 따른 인터프리터 행동 변화의 본질:
- **일반 메서드 (def method(self))**: 호출 시 `method(instance)`가 전달되어 `self`라는 변수에 인스턴스 객체가 담김.
- **@classmethod**: 호출 시 규칙 변경! `method(MyClass)`가 수행되어 인스턴스가 아닌 **클래스 자신(cls)**이 첫 번째 인자로 꽂힘. (객체 생성 전 호출 가능한 팩토리 메서드 구현의 핵심)
- **@staticmethod**: 아무것도 전달하지 않으며, 클래스 네임스페이스에 묶인 일반 함수처럼 동작함.

### 3. 상속(Inheritance)의 본질: Is-A vs Has-A
- **상속 ('Is-A' 관계)**: "전기차는 자동차이다." 부모의 모든 정체성과 계약을 계승함. 부모를 필요로 하는 모든 위치에 자식을 대체 투입할 수 있는 강력한 다형성을 제공함.
- **미상속 호출 ('Has-A' 관계 / Composition)**: "운전자는 자동차를 가진다." 클래스 간 결합도를 낮추기 위해 부품처럼 가져다 쓰는 방식이며, 일반적으로 상속보다 유연한 설계를 위해 권장됨.

## Ⅱ. 시스템 아키텍처 및 설계 단계

### [설계 6단계 프로세스]
1. **1단계 → 파일 구조**: `main.py`, `game.py`, `quiz.py`, `utils.py` 분리
2. **2단계 → 책임 분리**: SRP(단일 책임 원칙)에 따라 하나의 클래스는 하나의 기능 담당에 집중함.
3. **3단계 → 데이터 흐름**: JSON → 팩토리 로직 → RAM 적재 → 로직 실행 → 아토믹 저장.
4. **4단계 → 연결 관계**: `Quiz` 클래스의 의존성 차단을 위해 `from_dict` 팩토리 메서드 활용함.
5. **5단계 → 예외 케이스**: `FileNotFound`, `DecodeError` 등 파일 시스템의 모든 수호를 전담함.
6. **6단계 → 데이터 스키마**: `best_score`, `history`, `quizzes` 3대 키값 확정.

## Ⅲ. 세부 부품 명세

### 1. Quiz.py (Model)
- **구조**: `question`, `choices`, `answer` 속성 캡슐화.
- ** display / check_answer**: 자기가 가진 정답 번호를 기반으로 출력과 판정을 스스로 수행하는 정보 전문가 패턴 적용함.
- **to_dict / from_dict**: 영속성 연동을 위한 직렬화/역직렬화 전담 로직 탑재함.

### 2. Utils.py (Filter)
- **임무**: 빈 칸, 문자, 범위 밖 숫자 재입력 유도 작업 수행함.
- **비상 탈출**: `KeyboardInterrupt`, `EOFError` 감지 시 상위 계층으로 `raise`하여 세이브 시스템 트리거함.

### 3. Game.py (Brain & Persistence)
- **Atomic Write**: `.tmp` 파일을 이용해 저장 도중 전원 종료 시 원본 보존함.
- **JSON 예외 대응**: 
    - 파일 없음: `quizzes = []`, `score = 0` 초기화 작업.
    - 문법 훼손: `.bak` 백업 후 신규 생성으로 구동 강제함.
    - 내용 변질: `KeyError` 발생 시 해당 퀴즈 블록만 폐기하고 나머지는 최대한 구출함.

### 4. Main.py (Dome)
- **최종 방어 로직**: 앱 전체를 거대 `try-except`로 감싸 비정상 종료 시에도 RAM 데이터를 하드디스크로 즉시 대피시킴.

- **절차적 설계**: 1회 플레이 세션을 완전히 독립시켜 이전 세션의 오염을 남기지 않음.
- **무결 상태 초기화**: 에러 발생 시 즉시 처음부터 다시 실행 가능한 등가성(Idempotency) 확보함.

## Ⅵ. 요약 및 정리
1. **플레이 기준**: 모든 문제 진행을 1개의 플레이로 규정하되, '포기(5번)'는 정상 종료로 간주하여 점수를 합산함.
2. **점수 집계**: 실시간 RAM 스코어링 후 최종 결산소에서 최고 점수 비교 갱신 작업을 수행함.
3. **무결성 사수**: 비정상 강격 탈출 시에도 `main.py`에 설치된 최상위 Dome이 작동하여 기존의 모든 데이터를 안전하게 파일로 귀환시킴.
