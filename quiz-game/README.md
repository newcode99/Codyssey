# [Ultimate Project] 파이썬 전문 퀴즈 시스템 (Console Quiz Game)

본 문서는 파이썬 객체지향(OOP)의 근본 원리와 실무 수준의 시스템 견고함을 달성하기 위한 설계 철학을 집대성한 기술 명세이자 학습 일지임.

---

## Ⅰ. 객체지향(OOP) 설계 철학 및 기술 지식

### 1. Class: 설계도와 인스턴스
- **정의**: 데이터(멤버)와 기능(메서드)을 하나로 묶은 설계 도면. 클래스(Class)는 객체(Object)를 만들기 위한 **설계도나 틀**임.
- **클래스 vs 클로저**:
    - **클래스**: 설계도를 만들고 그 틀로 객체를 찍어내어 상태(self)를 영구적으로 기억함.
    - **클로저**: 함수 안에 변수를 숨겨두고, 그 변수를 사용하는 내부 함수만 꺼내어 쓰는 방식.
- **핵심 역할**:
    - 캡슐화 및 정보 은닉을 통해 외부의 부당한 조작으로부터 데이터를 보호함.
    - 동일 구조의 객체를 다수 생성하여 메모리를 효율적으로 관리함.

### 2. 클래스 메서드 (@classmethod) 동작 규칙
- **@classmethod**: 호출 시 규칙 변경! `method(MyClass)`가 수행되어 인스턴스가 아닌 **클래스 자신(cls)**이 첫 번째 인자로 꽂힘. 객체가 생성되지 않은 시점에도 JSON 데이터를 받아 스스로를 복원하는 팩토리 메서드 구현의 핵심임.

### 3. 상속(Inheritance)의 본질: Is-A vs Has-A
- **상속 ('Is-A' 관계)**: "전기차는 자동차이다." 부모의 정체성을 계승하는 강력한 계약임.
- **미상속 호출 ('Has-A' 관계 / Composition)**: "운전자는 자동차를 가진다." 클래스 간 결합도를 낮추기 위해 부품처럼 가져다 쓰는 방식이며, 본 게임은 `Game`이 `Quiz` 부품을 가지는 **Has-A** 구조를 채택함.

## Ⅱ. 4단계 기술 명세 (Technical Specs)

### 1단계: Quiz.py (Model)
- **역할**: 문제 정보(문제, 선택지, 정답)의 무결성을 지키는 데이터 객체.
- **행동 (Methods)**: 
    - `display()`: 터미널 환경에 최적화된 선택지 출력 로직.
    - `from_dict()`: 딕셔너리 데이터를 분해하여 객체로 조립하는 '의존성 차단'의 정수.

### 2단계: Utils.py (Filter)
- **역할**: 사용자로부터 오직 '허용된 정수'만을 정제해내는 검문소.
- **처리기**: 빈 칸, 문자 입력, 범위 밖 숫자 재입력을 유도하며 비상 신호(`raise`)를 상위로 전달함.

### 3단계: Game.py (Persistence & Brain)
- **책임**: 퀴즈 데이터의 생명주기(Life-cycle)와 영속성을 총괄함.
#### A. 데이터 (Attributes 및 경로 관리)
- **Self-Seeding**: 파일(`state.json`)이 없을 때 `FileNotFoundError`를 감지하여 하드코딩된 **'나라별 수도 퀴즈 5종'**을 자동으로 생성함. 이는 데이터가 없는 극한의 상황에서도 앱의 기능을 보장하기 위함임.
- **Path Locking**: 실행 위치(CWD)에 상관없이 항상 `quiz-game/state.json` 한 곳만 바라보도록 경로를 강제 고정함.
#### B. 행동 (영속성 로직)
- **Atomic Write**: `.tmp` 임시 파일을 사용하여 저장 도중 전원이 꺼져도 원본 데이터가 0바이트로 깨지는 현상을 물리적으로 방지함.
- **Corruption Recovery**: JSON 문법이 깨졌을 경우(`.bak`) 백업본을 만들고 시스템을 긴급 리셋하여 안정적인 재구동을 지원함.

### 4단계: Main.py (Dome & Entry Point)
- **책임**: 사용자에게 인터페이스를 제공하고, 시스템 전반의 비상 종료를 총괄 방어함.
#### A. 데이터 (입력 환경 최적화)
- **ReadLine CJK support**: `import readline`을 통해 한글(광폭 문자) 입력 시 발생하는 커서 위치 불일치 및 백스페이스 잔상 문제를 해결함. (입력 경험 UX 강화)
#### B. 행동 (최종 방어 로직)
- **Unified Exception Shield**: 앱 전체를 거대 `try-except`로 감싸 유저의 돌발적인 `Ctrl+C/D` 상황에서도 메모리에 머물던 데이터를 하드디스크로 즉시 대피(`save_state`)시키는 사후 조치를 수행함.
